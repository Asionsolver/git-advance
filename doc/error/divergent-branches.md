# ⚠️ Troubleshooting: Git Pull (Divergent Branches)

যখন আপনি `git pull` দিতে যান এবং গিট আপনাকে **"fatal: Need to specify how to reconcile divergent branches"** মেসেজটি দেখায়, তার মানে হলো আপনার লোকাল (Local) এবং রিমোট (Remote)—উভয় জায়গাতেই আলাদা আলাদা নতুন কমিট জমা হয়েছে। গিট এখন দ্বিধায় আছে যে এই দুটি ভিন্ন ইতিহাসকে সে কীভাবে একত্রে মেলাবে।

---

## 🔍 কেন এই সমস্যাটি হয়? (Divergence Explained)

সহজ কথায়, আপনার কোডবেস এখন দুটি ভিন্ন পথে বিভক্ত হয়ে গেছে:

1.  **রিমোটে (GitHub):** এমন কিছু পরিবর্তন আছে যা আপনার পিসিতে নেই।
2.  **লোকালে (পিসিতে):** এমন কিছু পরিবর্তন আছে যা আপনি এখনও গিটহাবে পাঠাননি।

> [!IMPORTANT]
> যেহেতু দুটি পথ আলাদা হয়ে গেছে, গিট আপনাকে একটি **"একত্রীকরণ পদ্ধতি" (Reconciliation Method)** বেছে নিতে বলছে যাতে ইতিহাস পুনরায় একীভূত করা যায়।

---

## 🛠️ সমাধানের ৩টি প্রধান পথ (The 3 Ways to Fix)

গিট আপনাকে তিনটি অপশন দিচ্ছে। আপনার কাজের ধরন অনুযায়ী যেকোনো একটি বেছে নিতে পারেন:

### ১. মার্জ পদ্ধতি (Merge - সবচেয়ে নিরাপদ)

এটি আপনার লোকাল এবং রিমোটের কাজগুলোকে মিলিয়ে একটি নতুন **"Merge Commit"** তৈরি করবে। নতুনদের জন্য এটিই সবচেয়ে সহজ এবং নিরাপদ পদ্ধতি।

**কমান্ড:**

```bash
git pull origin feat/login --no-rebase
```

### ২. রিবেস পদ্ধতি (Rebase - পরিষ্কার ইতিহাসের জন্য)

এটি আপনার লোকাল কমিটগুলোকে সাময়িকভাবে সরিয়ে রেখে রিমোটের কমিটগুলো আগে বসাবে, তারপর তার উপরে আপনার কাজগুলো জুড়ে দেবে। এতে প্রজেক্টের ইতিহাস একদম সোজা (Linear) থাকে।

**কমান্ড:**

```bash
git config pull.rebase true
git pull origin feat/login
```

### ৩. শুধুমাত্র ফাস্ট-ফরোয়ার্ড (Fast-forward only)

এটি তখনই কাজ করবে যদি আপনার লোকালে কোনো বাড়তি কমিট না থাকে। যদি লোকাল কমিট থাকে, তবে এটি এরর দেবে এবং আপনাকে অন্য পদ্ধতি নিতে বলবে।

**কমান্ড:**

```bash
git config pull.ff only
```

## 💡 আপনার জন্য সেরা সমাধান (Quick Fix)

আপনি যদি কোনো জটিলতা ছাড়া দ্রুত কাজ শেষ করতে চান, তবে নিচের ধাপগুলো অনুসরণ করুন:

1. **ধাপ ১ (ভবিষ্যতের জন্য ডিফল্ট সেট করুন):**
   ভবিষ্যতে সব প্রজেক্টের জন্য 'Merge' পদ্ধতি ডিফল্ট সেট করতে নিচের কমান্ডটি দিন:

   ```bash
   git config --global pull.rebase false
   ```

2. **ধাপ ২ (এখন পুল করুন):**

   ```bash
   git pull origin feat/login
   ```

3. **ধাপ ৩ (এবার নিরাপদে পুশ করুন):**
   ```bash
   git push origin feat/login
   ```

## 📊 মার্জ বনাম রিবেস (এক নজরে)

| বৈশিষ্ট্য            | মার্জ (`--no-rebase`)             | রিবেস (`--rebase`)                         |
| :------------------- | :-------------------------------- | :----------------------------------------- |
| **ইতিহাস (History)** | একটু জটিল (মার্জ কমিট দেখায়)।    | খুব পরিষ্কার এবং সোজা (Linear)।            |
| **নিরাপত্তা**        | খুবই নিরাপদ, তথ্য হারানোর ভয় নেই। | একটু অ্যাডভান্সড, সাবধানে ব্যবহার করতে হয়। |
| **ব্যবহার**          | টিম প্রজেক্টে বা সাধারণ কাজে।     | ব্যক্তিগত ফিচারে বা ক্লিন হিস্ট্রি চাইলে।  |

---

> [!TIP]
> **প্রো টিপ:** যদি পুল করার সময় **Merge Conflict** আসে, তবে ঘাবড়াবেন না। ফাইলগুলো ওপেন করে ম্যানুয়ালি কোড ঠিক করুন, তারপর `git add .` এবং `git commit` করে কাজ শেষ করুন।

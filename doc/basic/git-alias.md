আপনার দেওয়া তথ্যগুলোকে আরও প্রফেশনাল, দৃষ্টিনন্দন (Eye-catchy) এবং রিডেবল করার জন্য নিচে একটি স্ট্যান্ডার্ড Markdown (.md) গাইড ফরম্যাট দেওয়া হলো:

# 🛠️ Git Mastery: Power Up with Git Alias

**গিট অ্যালিয়াস (Git Alias)** হলো বড় এবং জটিল কমান্ডগুলোকে ছোট শর্টকাটে রূপান্তর করার একটি দুর্দান্ত উপায়। এটি আপনার ডেভেলপমেন্ট প্রসেসকে আরও দ্রুত এবং কার্যকর করে তোলে।

---

## 🔍 ১. গিট অ্যালিয়াস কী?

সহজ কথায়, গিট অ্যালিয়াস হলো আপনার নিজের তৈরি করা **কাস্টম শর্টকাট**। গিট আপনার কনফিগারেশন ফাইল থেকে এই শর্টকাটটি পড়ে এবং এর বিপরীতে থাকা মূল কমান্ডটি ব্যাকএন্ডে কার্যকর করে।

---

## 🛠️ ২. আপনার পিসিতে অ্যালিয়াস চেক করা

বর্তমানে আপনার কম্পিউটারে কোন কোন শর্টকাট সেট করা আছে তা দেখতে টার্মিনালে নিচের কমান্ডটি লিখুন:

```bash
git config --global --list
```

> **আউটপুট বিশ্লেষণ:** যদি আপনার আউটপুটে `alias.something=...` নামক কোনো লাইন না থাকে, তবে বুঝবেন আপনার এখনো কোনো অ্যালিয়াস সেট করা নেই। সাধারণত শুরুতে শুধুমাত্র ইউজার ইমেইল ও নাম সেটআপ করা থাকে।

---

## 🚀 ৩. নতুন অ্যালিয়াস সেটআপ করা (Expert Way)

ধরা যাক, আপনি `git log --graph --oneline` কমান্ডটিকে একটি ছোট শর্টকাট `git lg` তে রূপান্তর করতে চান।

**সেটআপ কমান্ড:**

```bash
git config --global alias.lg "log --graph --oneline"
```

**কিভাবে ব্যবহার করবেন?**
এখন থেকে আপনাকে আর বড় কমান্ড লিখতে হবে না। শুধু টার্মিনালে লিখুন:

```bash
git lg
```

_এটি আপনাকে সুন্দর গ্রাফিক্যাল ভিউ এবং এক লাইনে কমিট হিস্ট্রি দেখাবে।_

---

## 💡 ৪. জনপ্রিয় কিছু গিট অ্যালিয়াস (Recommended)

আপনার কাজকে আরও সহজ করতে নিচের অ্যালিয়াসগুলো সেটআপ করে নিতে পারেন:

| কাজ (Task)       | মূল কমান্ড (Full Command)         | শর্টকাট (Alias) | সেটআপ করার কমান্ড                                             |
| :--------------- | :-------------------------------- | :-------------- | :------------------------------------------------------------ |
| **Status**       | `git status`                      | `git s`         | `git config --global alias.s status`                          |
| **Commit**       | `git commit -m`                   | `git c`         | `git config --global alias.c "commit -m"`                     |
| **Checkout**     | `git checkout`                    | `git co`        | `git config --global alias.co checkout`                       |
| **Detailed Log** | `git log --graph --oneline --all` | `git lga`       | `git config --global alias.lga "log --graph --oneline --all"` |

---

## ⚠️ ৫. বিশেষজ্ঞ পরামর্শ (Expert Advice)

> [!IMPORTANT]
> **সুবিধা:** সময় সাশ্রয় হয় এবং টাইপিং এর ভুল (Typo) হওয়ার সম্ভাবনা কমে।
>
> **সতর্কতা:** একজন বিগিনার হিসেবে শুরুতে মূল কমান্ডগুলো (`git log`, `git status`) লিখে অভ্যস্ত হওয়া ভালো। কারণ, যখন আপনি অন্য কারো পিসিতে বা কোনো প্রোডাকশন সার্ভারে কাজ করবেন, সেখানে আপনার এই কাস্টম শর্টকাটগুলো থাকবে না।

---

## 🗑️ ৬. অ্যালিয়াস ডিলিট করার নিয়ম

যদি কোনো শর্টকাট আর প্রয়োজন না হয়, তবে সেটি নিচের কমান্ড দিয়ে সহজেই মুছে ফেলতে পারেন:

**উদাহরণ (lg অ্যালিয়াসটি মুছতে):**

```bash
git config --global --unset alias.lg
```
